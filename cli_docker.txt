Docker Hub

docker login -u <ชื่อผู้ใช้>
เผยแพร่อิมเมจไปยัง Docker Hub
docker push <ชื่อผู้ใช้>/<ชื่ออิมเมจ>

docker search <ชื่ออิมเมจ>

General Command
เริ่มการทำงาน Docker Daemon
docker -d
แสดงความช่วยเหลือเกี่ยวกับ Docker (สามารถใช้กับคำสั่งย่อยทั้งหมดได้)
docker --help
แสดงข้อมูลระบบทั้งหมด
docker info

Containers
คอนเทนเนอร์คืออินสแตนซ์ที่ทำงานจากอิมเมจ Docker โดยคอนเทนเนอร์จะทำงานเหมือนเดิมเสมอไม่ว่าโครงสร้างพื้นฐานจะเป็นอย่างไร คอนเทนเนอร์แยกซอฟต์แวร์ออกจากสภาพแวดล้อมและรับรองว่าซอฟต์แวร์จะทำงานเหมือนกันแม้จะมีสภาพแวดล้อมที่แตกต่างกัน เช่น ระหว่างการพัฒนาและการทดสอบ
คำสั่งพื้นฐานเกี่ยวกับคอนเทนเนอร์
สร้างและรันคอนเทนเนอร์จากอิมเมจ พร้อมกำหนดชื่อ:
docker run --name <ชื่อคอนเทนเนอร์> <ชื่ออิมเมจ>
รันคอนเทนเนอร์และเปิดพอร์ตของคอนเทนเนอร์ไปยังโฮสต์:
docker run -p <พอร์ตโฮสต์>:<พอร์ตคอนเทนเนอร์> <ชื่ออิมเมจ>
รันคอนเทนเนอร์ในโหมดแบ็กกราวด์:
docker run -d <ชื่ออิมเมจ>
เริ่มหรือหยุดคอนเทนเนอร์ที่มีอยู่:
docker start|stop <ชื่อคอนเทนเนอร์> (หรือ <container-id>)
ลบคอนเทนเนอร์ที่หยุดทำงานแล้ว:
docker rm <ชื่อคอนเทนเนอร์>
เปิดเชลล์ภายในคอนเทนเนอร์ที่กำลังทำงาน:
docker exec -it <ชื่อคอนเทนเนอร์> sh
ดึงและติดตามล็อกของคอนเทนเนอร์:
docker logs -f <ชื่อคอนเทนเนอร์>
ตรวจสอบข้อมูลคอนเทนเนอร์ที่กำลังทำงาน:
docker inspect <ชื่อคอนเทนเนอร์> (หรือ <container-id>)
แสดงรายการคอนเทนเนอร์ที่กำลังทำงาน:
docker ps
แสดงรายการคอนเทนเนอร์ทั้งหมด (ทั้งที่ทำงานและหยุดแล้ว):
docker ps --all
ดูสถิติการใช้ทรัพยากร:
docker container stats
